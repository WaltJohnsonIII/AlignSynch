## ðŸ“‹ **Phase 3: Spec-Kit Integration**

### **Step 3.1: Install Spec-Kit**
```bash
# Install spec-kit
npm install @github/spec-kit

# Initialize spec-kit
npx spec-kit init

# Verify installation
npx spec-kit --version
```

### **Step 3.2: Create Spec Structure**
```bash
# Create directory structure
mkdir -p specs
mkdir -p templates
mkdir -p .spec-workflow
mkdir -p .spec-workflow/approvals
mkdir -p .spec-workflow/tasks
mkdir -p .spec-workflow/steering

# Verify structure
tree specs/
tree templates/
tree .spec-workflow/
```

### **Step 3.3: Setup Templates**
```bash
# Copy templates
cp templates/spec-template.md specs/
cp templates/plan-template.md specs/
cp templates/tasks-template.md specs/
cp templates/constitution.md specs/

# Verify templates
ls -la specs/
ls -la templates/
```

### **Step 3.4: Configure Spec-Kit**
```bash
# Create spec-kit configuration
cat > .spec-kit.json << EOF
{
  "workflow": "spec-kit",
  "templates": "templates/",
  "specs": "specs/",
  "constitution": "templates/constitution.md"
}
EOF

# Create MCP configuration
cat > .mcp.json << EOF
{
  "servers": ["goose", "continue", "gemini"],
  "context": "project-specific",
  "autoConnect": true
}
EOF
```
